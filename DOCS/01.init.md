**Role:** Act as a Principal Java Backend Architect and Spring Boot Expert.

**Objective:** Scaffold a cutting-edge, production-ready Authentication REST API using **Spring Boot 4.0.2** and **Java 21 (LTS)**.

**Technical Stack:**

- **Language:** Java 21 (Use `records` for immutable DTOs and `sealed interfaces` for service results).
- **Framework:** Spring Boot 4.0.2 (Spring Framework 7, Spring Security 7).
- **Standards:** Jakarta EE 11 (Servlet 6.1, Persistence 3.2).
- **Database:** PostgreSQL (with `pgvector` or standard UUID support).
- **Security:** Stateless JWT Authentication + Argon2 or BCrypt.

**Context (Database Schema):**
Map the provided SQL schema to a Jakarta Persistence Entity (`User.java`).
_Note:_ The schema uses `uuid_generate_v4()`. Ensure the Entity uses `@GeneratedValue(strategy = GenerationType.UUID)` (available in Hibernate 6.x/7.x) rather than the generic `AUTO`.

**Task Requirements:**

1.  **Configuration (`application.properties`):**
    - Configure `spring.datasource.*` for PostgreSQL.
    - **New in Boot 4:** Enable **Structured Logging** (`logging.structured.format.console=ecs` or `json`) for better observability.
    - Set up JWT secrets using `${JWT_SECRET}` placeholders.

2.  **Modern Architecture & Validation:**
    - **Records:** Use Java Records for `LoginRequest` and `SignUpRequest`.
    - **Validation:** Use Jakarta Validation 3.1 (`@Email`, `@NotBlank`).
      - _Rule:_ Password must be Strong (min 8 chars, mixed case, special char) using a custom `@StrongPassword` annotation or `@Pattern`.
    - **API Versioning:** Use Spring Boot 4's native support (e.g., header-based or path-based `/api/v1/auth`).

3.  **Security (Spring Security 7):**
    - Implement the `SecurityFilterChain` bean.
    - **Crucial:** Explicitly handle the new **Lambda DSL** for configuration (as chained methods were deprecated/removed).
    - Stateless Session Management.

4.  **Endpoints:**
    - `POST /api/v1/auth/signup`: Handle duplication errors (Unique Email) specifically with a 409 Conflict.
    - `POST /api/v1/auth/login`: Return a `TokenResponse` record.

**Output Format:**

- `application.properties` (with structured logging enabled).
- `User` Entity (using Jakarta namespace).
- `SecurityConfig` (Spring Security 7 style).
- `AuthService` logic.
